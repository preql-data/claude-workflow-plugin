| Status | Meaning |
| ------ | ------- |
| **PROVEN** | An executed artifact demonstrates the claim — a test run, a config inspection whose semantics are unambiguous, a live probe, or a recorded trace. The evidence pointer resolves (file exists, run log cited, assertion count reproducible). |
| **PROVEN-WITH-CAVEAT** | The claim is true with a documented, cited caveat — e.g. true at the config/structural level while the end-to-end behavioral confirmation is pending, or true in substance while the doc wording overstates. The caveat is part of the row and survives into the release notes as residual risk. |
| **NOT-PROVEN** | No executed evidence today, OR the claim as written is contradicted by the current code/config (contradicted rows say so explicitly — they are prove-or-remove rewrite candidates, not silent failures). Every NOT-PROVEN row names the concrete mechanism that will flip it. |
| **REMOVED** | The claim was false or stale and has been removed/rewritten in the docs. The row records what it said, why it was false, and the commit/task that fixed it. |
